# Зоны интерфейса по ролям

Документ зафиксирован по матрице полномочий backend (`auth_role` / `auth_grant`, миграции V36–V39) и контроллерам `employee-progress-service`. При смене грантов на API обновлять эту таблицу.

Роли в JWT совпадают с полем `title` в `auth_role`:

| id | title (claim в токене) |
|----|-------------------------|
| 1000 | Директор отдела |
| 1001 | Генеральный директор |
| 1002 | Тимлид |
| 1003 | Сотрудник |

У роли **Генеральный директор** (1001) в БД `check_orgs = true`: набор экранов совпадает с **Директором отдела**, отличие — в объёме данных по оргструктуре (когда фильтрация реализована на backend).

Источник правды по API: `D:\javaprojects\employee-progress\employee-progress-service`, OpenAPI `http://localhost:8008/employee-progress/contract`.

## Доступ по ролям

Легенда:

- **полный** — все действия по грантам зоны;
- **просмотр** — только чтение;
- **ограниченно** — часть действий (см. текст в ячейке);
- **—** — нет гранта: пункт навигации скрывать или показывать заглушку «нет доступа».

| Зона | Ген. директор (1001) | Директор отдела (1000) | Тимлид (1002) | Сотрудник (1003) |
|------|----------------------|-------------------------|---------------|------------------|
| **А. Матрица** | **полный** (в т.ч. редактирование) | **полный** | **просмотр** матрицы и справочников | **просмотр** |
| **Б. Сотрудники и грейды** | **полный** (+ назначение грейда) | **полный** (+ назначение) | **просмотр** истории и текущего грейда | **просмотр** (объект списка — по правилам backend) |
| **В. Политики** | **полный** | **полный** | **просмотр** | **просмотр** |
| **Г. ИПР** | в основном **наблюдение** + вложения; без черновика задач и без смены статуса плана | то же | **рабочая зона**: создание ИПР, черновик задач, обновление плана, смена статуса плана, удаление вложений, апрув компетенций тимлидом, запись прогресса; смена статуса задачи (совместно с сотрудником по грантам API) | **«Мои ИПР»**: просмотр, прогресс, вложения, комментарии, смена статуса задачи где разрешено; **нет** создания плана и черновика задач |

## Дополнительные правила UI

1. **Генеральный директор и директор отдела** — одинаковый набор зон **А–Е** с точки зрения прав; различие — орг. охват данных (когда поддержан на backend).

2. **Тимлид** — основная зона **Г (ИПР)** и операционная **Д (ревью)**; отчёты без истории кадровых решений.

3. **Сотрудник** — фокус на **Г**; административные зоны **А (редактирование)**, **Б (назначение)**, **В (управление)**, **Е**, **Ж** недоступны по грантам.

4. **Зарплатная вилка** в зоне **А**: для роли сотрудника не отображать чувствительные суммы (см. `D:\javaprojects\employee-progress\employee-progress-service\docs\database-schema-reference.md`, §6).

5. Список authority для кнопок и запросов держать в синхроне с `OuGrantSpelExpressions` и миграциями грантов; при 403 не считать ошибкой клиента без сообщения с сервера.
